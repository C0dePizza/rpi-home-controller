RPi Home Controller

Material Design based web-app that controls your home.

View project on GitHub

Overview

The RPi Home Controller is a simple, yet useful set of scripts allowing the control and interaction with the GPIO of your Pi. The web interface is built using the Materialize css framework. http://materializecss.com/

Useful links

Super easy to use 110v 'relay' here.

Or if you want to do it yourself, this Instructable is a great way to wire up your home.

Features

  • Aesthetically appealing web interface
  • Synchronization across web clients
  • An additional TCP socket server (for easy integration with third party applications)
  • Fully customizable display
  • And much, much more...

Install Dependencies

# pip install autobahn twisted jinja2

Usage

$ git clone https://github.com/therealtylerdurden/rpi-home-controller.git

$ cd rpi-home-controller

$ mv sample-data.json data.json   # Move config file if you just cloned

# ./main.py

Or if you want to detach process, run in background, and log output:

# nohup ./main.py &

Configuration

The configuration is located in data.json

Most of it is relatively self-explanatory, however, configuring groups needs to be explained:

"Web": {
    "Groups": [
        {
            "description": "AC",    // "description" is the label displayed above the switch
            "gpiopin": "40"    // "gpiopin" is the pin on the Pi that will be toggled
        },
    ]
}

API

The API is very easy to use, and the syntax is the same for both the WebSocket server and the TCP server.

PIN=<pin number>,OUT,<1,0> - This sets the pin high or low (1 or 0)

PIN=<pin number>,IN,0 - This returns the current state of the pin. The TCP server will return 1 or 0, but the WebSocket server will return PIN <pin number> <1,0>

ALLRELAYS=<1,0> - This command will read data.json and toggle all the pins defined either high or low
